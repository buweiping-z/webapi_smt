# 多照片点检 — 设计文档

> 日期：2026-06-19 | 状态：已确认

## 目标

支持每个点检异常项拍摄多张照片（上限 5 张），覆盖两个场景：
- 填表时连续拍多张（批量拍 → 提交时自动上传）
- 提交后补拍追加

## 现状分析

| 层 | 多照片支持 |
|---|---|
| `inspection_photos` 表 | ✅ 无唯一约束，`photo_order` 字段已存在 |
| `PhotosController` API | ✅ 上传/查询/删除均支持多张 |
| Web 前端 Lightbox | ✅ 已支持前后翻页和计数 |
| Web 月视图表格 | ✅ 异常格显示 `📷N`，点击查看 |
| Android 端 | ❌ `photoLocalPath: String?` 仅单张 |

## 方案选型

选择 **方案 A：最小改动**。

- Android 端：`photoLocalPath: String?` → `photoLocalPaths: List<String>`（最多 5）
- 批量拍摄，提交时自动串行上传
- 后端 + Web 端几乎不动

## 详细设计

### 1. Android 数据模型变更

**文件：** `InspectionViewModel.kt`

```
// 当前
data class InspectionItemState(
    ...
    val photoLocalPath: String? = null,
    val isPhotoUploading: Boolean = false,
    val photoUploaded: Boolean = false
)

// 改为
data class InspectionItemState(
    ...
    val photoLocalPaths: List<String> = emptyList(),  // 最多 5 张
    val isPhotoUploading: Boolean = false,
    val photoUploadedCount: Int = 0,  // 该项已上传张数
)
```

`PendingPhotoItem.missingItems` 从 `List<String>`（仅 itemName）改为带 `existingPhotoCount` 的结构体。

### 2. Android UI 变更

**文件：** `InspectionScreen.kt`

**填表阶段（Phase 1）— InspectionItemCard：**

- `requirePhoto` 项目显示缩略图网格（每行 3 张，80×80dp），每张右上角 ✕ 删除
- 未满 5 张时显示 `📷 拍照（N/5）` 按钮
- 满 5 张时按钮置灰：`已拍满 5 张`
- 无照片时不显示缩略图区域

**提交后（Phase 2）— 顶部 banner：**

- 提交时已有照片的项自动上传，Phase 2 banner 通常只用于补拍完全没拍的项
- banner 逻辑基本不变

### 3. 提交时自动上传

**文件：** `InspectionViewModel.kt` — `submitInspection()`

```
提交文字结果 → 收到 pendingPhotoItems
  → 遍历每个 missingItem：
      如果 photoLocalPaths 非空 → 串行上传，带 photoOrder=0,1,2...
  → 全部完成后检查：
      所有 missingItems 都有 ≥1 张上传？→ 直接完成
      否则 → banner 提示补拍缺失项
```

关键点：
- 串行上传（一张一张），避免文件锁/并发问题
- 进度显示「X/Y 张上传中」

### 4. 后端微调

**文件：** `Controllers/InspectionController.cs` — `records/save` 返回体

```json
// 当前
{ "pendingPhotoItems": [{"recordId": 110, "missingItems": ["温度"]}] }

// 改为
{ "pendingPhotoItems": [{"recordId": 110, "missingItems": [
    {"itemName": "温度", "existingPhotoCount": 0},
    {"itemName": "压力", "existingPhotoCount": 2}
]}]}
```

Android 端据此区分「完全没拍」和「已有 N 张」。

### 5. Web 端

**不做改动。** 当前表格内 `📷N` + Lightbox 前后翻页已满足需求。

## 不改动的部分

- `inspection_photos` 表结构
- `PhotosController.UploadPhoto` — 已接受 `photoOrder`
- `PhotosController.GetMonthlyPhotos` — 已按 `{recordId}_{itemName}` 分组返回数组
- `PhotosController.DeletePhoto` — 单张删除不变
- Web 端 HTML/CSS/JS

## 边界情况

- **重检场景：** `records/save` 已有逻辑删除旧照片 + 标记 pending，多张场景同样适用
- **删除单张：** 保持当前 API，每张独立删除；Android 端缩略图 ✕ 调用 `DELETE /api/inspection/photos/{id}`
- **网络失败：** 串行上传中某张失败，跳过继续，最后汇总失败张数提示重试
- **超出 5 张：** UI 按钮置灰阻止，不依赖后端校验
